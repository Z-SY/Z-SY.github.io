# 0x00 开篇

  • 勉励语 - 君子耻其言而过其行

# 0x01 基本符号

不贴图了，还得 P（狗头保命🤡）

  1. ⚪- 表示加工 - 在圆内说明加工的名字和编号
  2. →- 表示数据流 - 在箭头边给出数据流的名称和编号
  3. 两个下划线，一进一出的箭头 - 表示数据存储文件 - 文件名称为名词或名词性短语
  4. 长方形 - 表示数据源点或汇点 - 在方框内注明源点或汇点的名称

# 0x02 设计原则

  1. 自外向内，自顶向下，逐层细化，完善求精
  2. 保持父图和子图的平衡
  3. 保持数据守恒 - 任何的加工要有输入和输出
  4. 加工细节隐蔽
  5. 简化加工间的关系
  6. 均匀分解
  7. 适当地为数据流、加工、文件、源 / 宿命名，名字应反应该成分的实际意义，避免空洞的名字
  8. 忽略细枝末节
  9. 表现的是数据流而不是控制流

# 0x03 数据字典

  1. “=”- 代表被定义为
  2. “+”- 代表与，例 “x=a+b” 代表 x 是由 a 和 b 组成
  3. “[... ，...] 或 [... | ...]”- 代表或，例 “x=[a , b]” 代表 x 由 a 或 b 组成
  4. “{...}”- 代表重复，例 “x={a}” 代表 x 由 0 个 a 或者多个 a 组成
  5. “(...)”- 代表可选，例 “x=(a)” 代表 a 可在 x 中出现也可以不出现
