# 0x01 上帝之手

  • 计算机中，当我们找到我们关注的数据的地址，在过掉程序保护的前提下，我们可将数据修改为我们想要的数据，例如游戏中的金钱、换弹速度、弹药量等等，通过外部的程序修改的方式达到我们想要的目的，例如，我将换弹速度改为 1，可实现秒级换弹，将弹药量通过循环锁死，就可实现无限子弹的效果，总而言之，所有的计算机程序，说白了就是数据 + 算法，数据一般代表我们期待的输入，算法协助我们怎样更好的操纵数据，帮助我们得到想要的输出，因此，我认为的 game hack 方式，一个是直接修改数据，也就是我们期待的输入；另一个就是打补丁，修改操作数据的方式，例如本来开枪应该执行子弹数减一的操作，但我通过逆向修改了二进制代码或 hook 了相关函数执行流程得以略过减一操作，则也实现了无限子弹的效果。

# 0x02 Windows 的 API：RPM-WriteProcessMemory

  • RPM-WriteProcessMemory - 用来写入某一进程的数据，根据 MSDN 官网文档简介，参数如下
    1. 参数 - 1-hProcess：A handle to the process memory to be modified. The handle must have PROCESS_VM_WRITE and PROCESS_VM_OPERATION access to the process
    2. 参数 - 2-lpBaseAddress：A pointer to the base address in the specified process to which data is written. Before data transfer occurs, the system verifies that all data in the base address and memory of the specified size is accessible for write access, and if it is not accessible, the function fails
    3. 参数 - 3-lpBuffer：A pointer to the buffer that contains data to be written in the address space of the specified process
    4. 参数 - 4-nSize：The number of bytes to be written to the specified process
    5. 参数 - 5-lpNumberOfBytesWritten：A pointer to a variable that receives the number of bytes transferred into the specified process. This parameter is optional. If lpNumberOfBytesWritten is NULL, the parameter is ignored
